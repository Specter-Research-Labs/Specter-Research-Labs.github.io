DeepSeek Provider Settings

Model

l3lab/ntp-mathlib-context-deepseek-coder-1.3b (1.3B), loaded from a local MLX-converted checkpoint.

Current Implementation

DeepSeekTacticProvider currently uses stochastic sampling, not beam search.

Parameter Current (code) Notes
generation mode sampling mlx_lm.sample_utils.make_sampler(...)
temperature 0.6 set in provider sampler
top_p 0.9 set in provider sampler
max_new_tokens 64 per generation call
per-request sample count min(n, self._num_samples) default self._num_samples = 10

Relevant implementation: prover/providers/deepseek.py.

Why This Matters

  • This provider is optimized for candidate diversity in tactic suggestions.
  • Sampling behavior affects reproducibility and exploration variance.
  • Any analysis comparing providers should treat DeepSeek outputs as stochastic unless seeds and execution context are controlled externally.

Relation to miniCTX Defaults

miniCTX commonly uses deterministic decoding/beam settings; we currently use sampling here.

When changing decoding behavior, update both:

  • provider code (prover/providers/deepseek.py)
  • this document and docs/providers/tactic-provider-options.md
